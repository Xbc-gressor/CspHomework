这世界真小

今天参加公司ERP项目升级评标会，意外发现前来讲标的实施方团队里面有一个顾问恰好是之前做项目合作过的实施顾问，只不过那时候Ta是在另一家实施公司。转眼已经是好几年前了，不得不感叹这世界真小！

Ta自然没有认出我来，如果不是看到标书里Ta的项目简介，我也差点没认出Ta。在上一次的项目里，我是甲方内部顾问，Ta是实施方。刚开始中间还隔着一个团队叫“甲方代表”，言下之意这个团队就是代表甲方关键用户跟实施方谈需求的。到我们进入这个项目的时候才发现这个实施方基本上是脱离了甲方的内部团队在做项目了，而这个“甲方代表”实际上并不是完全独立的第三方，而是跟实施方有一定的关系背景。


可想而知这个项目做得有多离谱，从头到尾，Ta所在的实施方并不怎么爱搭理甲方团队，不少需求都是凭空想象和理所当然，不跟关键用户确认清楚，就连后面做系统交接的时候也没有什么交付物，美言之曰：没有合同规定需要文档输出，ERP系统配置为商业机密。如此下来，不仅内部IT同事感到不满，就连关键用户也都对他们怨气连天。不能好好交流，无法顺畅沟通以及态度强势是他们最大的特色标签。


我记得有一次用户跟他们反馈说ERP界面功能比较难用，看能否进行效率优化，结果他们直接回复说：此功能无法改善，请自行找德国ERP原厂商协商解决。对甲方关键用户的诉求基本爱理不理，能推就推，对甲方IT的要求也直接忽略，甚至投诉到高层，搞得甲方IT团队特别被动，根本无法一起做项目。ERP系统标准表数据更是随意更改，开发不规范，Bug一大堆，配置信息乱七八糟，整个实施过程并没有完整的文档记录。


后来偶尔一次看到网上关于ERP顾问的黑名单，一查才发现Ta们团队里面有几个榜上有名（没有Ta），怪不得这么招人反感。不过实话说，Ta本身的技术和业务能力还是很过关的，值得肯定，就是沟通交流上有待加强，跟其他团队和用户相处上也远远不够，性格强势，不苟言笑，有距离。


我经常说一个优秀的ERP顾问有三大评判标准：1、系统方案能力；2、业务分析能力；3、沟通交流及人际关系处理能力；尤其是第三点，更是一个优秀的顾问必备的品质。系统能力再差，业务分析能力较弱都可以通过练习和做项目慢慢改善弥补，而沟通能力、人际关系处理能力就不是时间和做项目能够堆积得了的。老祖宗说的江山易改本性难移，这么多年过去了，Ta的讲话也还能看出一点磕绊，像是在念稿背书，内容也更多是坐落在系统画面和功能等细节之上。



这个世界很小，且行且珍惜，说不定哪天就遇见。愿大家都能成为人见人爱的优秀顾问，系统可以不懂，业务能力可以不扎实，但沟通一定要友好，见面还能点头。
